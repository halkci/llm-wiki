# SCHEMA — 知识库运行规则

这份文档定义本知识库的结构与维护规则。**所有对 `wiki/` 的改动都必须遵守这里的规定。**

## 三层架构

1. **`sources/`** — 原始资料层,**不可变**。用户放入的 PDF、网页保存、笔记、图片、录音转写等。LLM 只读取,绝不修改或删除。
2. **`wiki/`** — 知识层,由 LLM 维护的 Markdown 页面。每页聚焦一个概念/主题/实体,持续被新源材料增量更新。
3. **`SCHEMA.md` / `INDEX.md` / `LOG.md`** — 元层,控制系统行为与导航。

## 页面命名

- 文件名使用 `kebab-case.md`,英文或拼音,不含空格。
- 中文主题的文件名用简短英文/拼音,页面标题用中文。
- 放在 `wiki/` 根目录即可,除非某主题膨胀到 >8 篇再考虑子目录。

## 页面结构

每个 wiki 页面必须以 YAML frontmatter 开头:

```markdown
---
title: 页面中文标题
aliases: [别名1, 别名2]
tags: [tag1, tag2]
sources:
  - sources/some-file.pdf
  - https://example.com/article
related: [[other-page]], [[another-page]]
created: 2026-04-05
updated: 2026-04-05
---

# 页面中文标题

## 摘要
一段话概括这个主题是什么、为什么重要。

## 核心内容
... 按小节组织,合成自多个源的信息 ...

## 与其他主题的关系
- 与 [[other-page]] 的联系:...

## 来源引用
- [^1]: sources/some-file.pdf, 第 3 节
- [^2]: https://example.com/article
```

## 交叉引用

- 页面间链接统一用 `[[page-slug]]`(不含 `.md` 后缀)。
- 每次更新一个页面时,**必须检查是否需要在相关页面添加反向链接**。
- 源引用用脚注 `[^n]` 指向 `sources/` 下的具体文件或外部 URL。

## 创建新页 vs. 更新已有页

**更新已有页**(默认):新信息属于已有主题,或与已有页有 >60% 重合。
**新建页**:出现一个明显独立的、未被覆盖的概念/实体/事件。新建时必须:
1. 在 `INDEX.md` 中登记。
2. 至少与一个已有页建立 `related` 链接。
3. 在 `LOG.md` 追加一行。

## INDEX.md(内容索引)

按主题分类列出所有 wiki 页面,是人类用户浏览知识库的入口。每次新建或重命名页面都要同步更新。

## LOG.md(变更日志)

按时间倒序记录每次 ingest/update/lint 的动作。格式:
```
## 2026-04-05
- [ingest] sources/foo.pdf → 新建 wiki/bar.md, 更新 wiki/baz.md
- [lint] 修复孤儿页 wiki/qux.md,添加到 INDEX
```

## 三种操作

### Ingest(摄入)
1. 读取新源材料(用户指定的 `sources/` 下文件或 URL)。
2. 提取关键信息:事实、论点、实体、关系。
3. 扫描现有 wiki 判断:哪些页面需要更新,是否需要新建页。
4. 执行更新;维护交叉引用。
5. 同步 INDEX.md 和 LOG.md。
6. **向用户汇报**:改动了哪些页面、新增了什么、是否发现冲突。

### Query(查询)
1. 先搜索 `wiki/`(这才是合成后的知识)。
2. 只有当 wiki 缺失或不充分时,才回退到 `sources/`。
3. 给出答案时标明依据的 wiki 页面 + 源引用。
4. **如果答案本身有价值**,询问用户是否把它沉淀成一个新 wiki 页或并入已有页。

### Lint(健康检查)
定期运行,检查并修复:
- **孤儿页**:未被 INDEX 收录或无任何入链。
- **死链**:`[[...]]` 指向不存在的页。
- **冲突**:不同页面对同一事实的矛盾陈述(标注出来,交用户裁决)。
- **空洞**:INDEX 登记但内容极少的占位页。
- **重复**:高度重合的页面(建议合并)。
- **过期**:`updated` 距今 >90 天且源材料有新版本的页面。

## 铁律

- **永不修改 `sources/`**。只读。
- **永不捏造来源**。每条断言都要能追溯到 `sources/` 或外部 URL。
- **冲突不隐藏**。发现矛盾就在页面里显式标注 `> ⚠️ 冲突:...` 并在 LOG 记录。
- **宁可让用户决定,不要擅自合并**。涉及合并/删除页面时先询问。
